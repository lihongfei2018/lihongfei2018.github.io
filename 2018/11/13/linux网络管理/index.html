netstat命令

显示网络连接

netstat–Ieth0
显示路由表：netstat{–route|-r} [–numeric|-n]
netstat -r 显示内核路由表
netstat -n 数字格式
相关选项：
-t: tcp协议相关
-u: udp协议相关
-w: raw socket相关
-l: 处于监听状态
-a: 所有状态
-n: 以数字显示IP和端口
-e：扩展格式
-p: 显示相关进程及PID
常用组合为 -nt -nta -nu -tnl

ip命令

配置Linux网络属性：ip命令
ip link 显示当前网络的链路信息
ip link set eh0 down 关闭当前网络链路的一个网卡
ip link set eh0 up 开启当前网络链路的一个网卡
ip addr简写ip a显示链路信息和网络层的信息
ip a addr 1.1.1.1/24 dev eth0给eth0网卡再增加一个IP地址
ip a del 1.1.1.1/24 dev eth0删除IP地址
ip a flush dev eht0清空网卡的IP信息
ip a addr 1.1.1.1/24 dev eth0 label eth2 新加一个IP地址，加别名
ip a a 1.1.1.1/24 dev ens33 scope (global|link|host)scope可以指明作用的区域，global：全局可用；link：仅当前网卡的链路可用，其他网卡进来的数据无法访问；host：仅本机可用。
ip route 显示路由信息
ip route add 添加路由
ip route del 删除路由
具体格式iproute add TARGET via GW dev IFACE srcSOURCE_IP
上面命令都是临时修改的，重启之后会恢复原样

ss命令

netstat通过遍历proc来获取socket信息，ss使用netlink与内核tcp_diag模块通信获取socket信息。
选项：
-t: tcp协议相关
-u: udp协议相关
-w: 裸套接字相关
-x：unixsock相关
-l: listen状态的连接
-a: 所有
-n: 数字格式
-p: 相关的程序及PID
-e: 扩展的信息
-m：内存用量
-o：计时器信息
常用组合：
-tan, -tanl, -tanlp, -uan
常见用法
ss-l 显示本地打开的所有端口
ss-pl显示每个进程具体打开的socket
ss-t -a 显示所有tcpsocket
ss-u -a 显示所有的UDP Socekt
ss-o state established ‘( dport= :sshor sport = :ssh)’ 显示所有已建立的ssh连接
ss-o state established ‘( dport= :http or sport = :http )’ 显示所有已建立的HTTP连接
ss-s 列出当前socket详细信息

网络配置文件

IP、MASK、GW、DNS相关配置文件：
/etc/sysconfig/network-scripts/ifcfg-IFACE
路由相关的配置文件：
/etc/sysconfig/network-scripts/route-IFACE
配置文件格式：
DEVICE：此配置文件应用到的设备
BOOTPROTO：激活此设备时使用的地址配置协议，常用的dhcp, static, none, bootp
IPADDR：指明IP地址
NETMASK：子网掩码
GATEWAY: 默认网关
DNS1：第一个DNS服务器指向是否生效看/etc/resolv.conf
DNS2：第二个DNS服务器指向
HWADDR：对应的设备的MAC地址
ONBOOT：在系统引导时是否激活此设备
TYPE：接口类型；常见有的Ethernet, Bridge
UUID：设备的惟一标识
USERCTL：普通用户是否可控制此设备
设备配置被保存在文本文件中
/etc/sysconfig/network-scripts/ifcfg-
动态配置格式
DEVICE=ethXHWADDR=0:02:8A:A6:30:45
BOOTPROTO=dhcp
ONBOOT=yes
Type=Ethernet
静态配置
DEVICE=ethX
HWADDR=0:02:8A:A6:30:45
IPADDR=192.168.0.123
NETMASK=255.255.255.0
GATEWAY=192.168.0.254
ONBOOT=yes
Type=Ethernet

网络接口配置-bonding（多网卡绑定）

将多块网卡绑定同一IP地址对外提供服务，可以实现高可用或者负载均衡。直接给两块网卡设置同一IP地址是不可以的。通过bonding，虚拟一块网卡对外提供连接，物理网卡的被修改为相同的MAC地址

Mode 0 (balance-rr)工作模式

轮转（Round-robin）策略：从头到尾顺序的在每一个slave，网卡轮流工作

Mode 1 (active-backup)工作模式

活动-备份（主备）策略：只有一个slave被激活，如果其中一个网卡down了，那么另一个网卡瞬间工作，为了避免交换机发生混乱此时绑定的MAC地址只有一个外部端口上可见。

Mode 3 (broadcast)工作模式

广播策略：在所有的slave接口上传送所有的报文,提供容错能力，如果有数据发送过来，两个网卡一块处理数据。

Bonding配置

创建bonding设备的配置文件
/etc/sysconfig/network-scripts/ifcfg-bond0文件名字必须是bond，后面数字可以自己指定，bonding的IP地址必须是和物理网卡在同一网段中。
DEVICE=bond0
IPADDR=
PREFIX=
BONDING_OPTS=“miimon=100 mode=0”
BOOTPROTO=none
bonding创建好之后，物理网卡也需要更改
DEVICE=eth0
MASTER=bond0
SLAVE=yes
USERCTL=no
BOOTPROTO=none
查看bond0状态：/proc/net/bonding/bond0可以查看是哪几个物理网卡是bonding，那个网卡在生效。

例：创建一个bonding 模式为1

1、创建一个文件,写入bonding文件格式,开始之前停止NetworkManager服务，并且防火墙需要时关闭状态，不然无法测试连通性。

2、创建或者修改物理网卡的信息

3、到此就已经创建好bonding模式1了，可以在/proc/net/bonding/bond0下查看相关信息。

4、卸载bonding，把配置文件删除，物理网卡的配置改为正常配置，用rmmod卸载bonding模块即可。
ifconfig bond0 down
rmmod bonding
