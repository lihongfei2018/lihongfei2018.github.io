linux文件查找

locate

locate是根据数据库来查找的，locate数据库相当于inde表，但是新建的文件locate数据库不会实时更新，所有新建的文件会搜索不到，更新数据库可以用updatedb来实现
locate查找比较模糊，只要包含搜索的字符串都会显示

工作特点:
•查找速度快
•模糊查找
•非实时查找
•搜索的是文件的全路径，不仅仅是文件名
•可能只搜索用户具备读取和执行权限的目录
有用的选项

-i不区分大小写的搜索
-n N只列举前N个匹配项目
-r 使用正则表达式

find

实时查找工具，通过遍历指定路径完成文件查找
工作特点：
•查找速度略慢
•精确查找
•实时查找
•可能只搜索用户具备读取和执行权限的目录

语法：
find [OPTION]… [查找路径] [查找条件] [处理动作]
查找路径：指定具体目标路径；默认为当前目录
查找条件：指定的查找标准，可以文件名、大小、类型、权限等标准进行；默认为找出指定路径下的所有文件
处理动作：对符合条件的文件做操作，默认输出至屏幕

###根据层级才搜索
如果直接执行find命令会直接显示当前工作目录下的所有文件全部显示，包括隐藏的文件,如果不加后面要搜索的内容，会把/data/下的所有目录文件显示一遍

最多搜索1级目录，是当前目录之后最多在1级目录

先处理目录内的文件，再处理目录-depth

根据名称来搜索

-name “文件名称”：支持使用glob "* *" 模糊查找
-iname”文件名称”：不区分字母大小写
-inumn 按inode号查找
-links n 链接数为n的文件
-regex 以PATTERN匹配整个文件路径，而非文件名称

根据属主、属组查找

-user USERNAME：查找属主为指定用户(UID)的文件
-group GRPNAME: 查找属组为指定组(GID)的文件
-uidUserID：查找属主为指定的UID号的文件
-gidGroupID：查找属组为指定的GID号的文件
-nouser：查找没有属主的文件
-nogroup：查找没有属组的文件

根据文件类型查找：

-type TYPE:
•f: 普通文件
•d: 目录文件
•l: 符号链接文件
•s：套接字文件
•b: 块设备文件
•c: 字符设备文件
•p: 管道文件
：-empty 空文件
find /app -type d -empty 目录

与或非

与：-a
或：-o
非：-not, !

德·摩根定律：
(非A) 或(非B) = 非(A 且B)
(非A) 且(非B) = 非(A 或B)

排除目录不搜索

-path…. -a –prune

根据文件大小查询

-size
find搜索文件并不是特别精确，比如说10M的文件，它搜索出来的是（9M-10M）之间的文件，相当于搜索内容-1 ，如果是以K为单位的话也是-1K；
比如搜索10M以下的文件，-10M 就是10M一下，但是不包括10M ，10M以上的话 +10

根据时间查询

根据时间查找也是和文件大小查询类似，比如说查询10天前修改过的文件，那么就是10天到11天之间的文件，也可以指定分钟
以“天”为单位 -atime -mtime -ctime 以“分钟”为单位-amin -mmin -cmin
访问时间，atime ; 修改时间, mtime ; 改变时间, ctime

根据权限查找

-perm 精确匹配 必须符合全部权限才会显示
-perm \777 只要有一个权限满足就会显示，任何一类用户(u,g,o)的权限中的任何一类(r,w,x)符合条件就满足，9位权限之间是或关系
-MODE：(u,g,o)的权限每一类同时符合则，满足条件，9位权限之间存在与关系每一类用户

处理动作

find传递查找到的文件至后面指定的命令的时候，查找到所有符合条件的文件一次性传递给后面的命令.
-print：默认的处理动作，显示至屏幕
-ls：类似于对查找到的文件执行“ls -l”命令
-delete：删除查找到的文件
-fls file：查找到的所有文件的长格式信息保存至指定文件中
-ok COMMAND {} \; 对查找到的每个文件执行由COMMAND指定的命令，对于每个文件执行命令之前，都会交互式要求用户确认
-exec COMMAND {} \; 对查找到的每个文件执行由COMMAND指定的命令
{}: 用于引用查找到的文件名称自身
find传递查找到的文件至后面指定的命令时，查找到所有符合条件的文件一次性传递给后面的命令
